# component.yml Patterns

`.component.yml` files in `packages/twig/components/` document the prop schema for a component. They are authored independently — they are not synced from the SDC package and not generated by any build script.

Always generate one. It is the authoritative prop schema for the twig package and the source material for any future SDC promotion.

---

## Schema

```yaml
name: Component Name
status: stable
description: One sentence describing what the component does.
props:
  type: object
  properties:
    theme:
      type: string
      enum: [light, dark]
    vertical_spacing:
      type: string
      enum: [top, bottom, both, none]
    with_background:
      type: boolean
    # component-specific props go here
    modifier_class:
      type: string
    attributes:
      type: "Drupal\\Core\\Template\\Attribute"
      title: HTML Attributes
```

**Do not include `$schema`.** The Drupal SDC validator URI (`https://git.drupalcode.org/project/drupal/-/raw/HEAD/core/assets/schemas/v1/metadata.schema.json`) is a Drupal-specific coupling and does not belong in the agnostic twig package. Any existing twig-package files that include it are erroneous.

---

## Standard props

Always include unless the component genuinely doesn't support them:

| Prop | Type | Notes |
|---|---|---|
| `theme` | `string` | enum: `[light, dark]` |
| `vertical_spacing` | `string` | enum: `[top, bottom, both, none]` — omit for atoms that never use spacing |
| `with_background` | `boolean` | omit for atoms |
| `modifier_class` | `string` | always include |
| `attributes` | `Drupal\Core\Template\Attribute` | always include |

---

## Enum values

These enum values are validated by `npm run validate`. Use exactly these values — no others.

| Prop | Valid values |
|---|---|
| `theme` | `light`, `dark` |
| `vertical_spacing` | `top`, `bottom`, `both`, `none` |

---

## Prop types

| Twig type | YAML type |
|---|---|
| string | `string` |
| boolean | `boolean` |
| array of objects | `array` with `items: type: object` |
| Drupal attributes | `Drupal\Core\Template\Attribute` |

For array props, list required child properties:

```yaml
items:
  type: array
  items:
    type: object
    required:
      - title
      - url
    properties:
      title:
        type: string
      url:
        type: string
      is_external:
        type: boolean
```

---

## Keeping docblock and .component.yml in sync

The `{# docblock #}` in the `.twig` file and the `.component.yml` describe the same props. In the SDC-first workflow `components-update-sdc.js` keeps these in sync automatically. In this twig-first workflow, both are written by hand — ensure every prop in `.component.yml` appears in the twig docblock and vice versa.
